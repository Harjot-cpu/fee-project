Operating Systems

Process Management

A process is a program in execution. The OS manages all running processes using a data structure called the Process Control Block (PCB), which stores the process ID, state, program counter, CPU registers, memory limits, and I/O status.

Process states include: New → Ready → Running → Waiting → Terminated. When the CPU switches from one process to another, the OS saves the current process state and loads the next — this is called a context switch.

PCB Structure: - Process ID (PID) - Process State (New, Ready, Running, Waiting, Terminated) - Program Counter (PC) - CPU Registers - Memory Management Info - I/O Status Information - Accounting Information

Threads are lightweight processes that share the same memory space. Multi-threading improves CPU utilization and responsiveness.

Operating Systems

CPU Scheduling Algorithms

The CPU scheduler selects a process from the ready queue and allocates the CPU. Key performance metrics include:

CPU Utilization (maximize), Throughput (maximize), Turnaround Time (minimize), Waiting Time (minimize), Response Time (minimize).

FCFS (First Come First Served): Processes executed in arrival order. Non-preemptive. Simple but can cause convoy effect. SJF (Shortest Job First): Process with smallest burst time first. Optimal average waiting time, but needs future burst time. Round Robin (Time Quantum = q): Each process gets q ms of CPU time. Preemptive. Good for interactive systems. Larger q → FCFS. Smaller q → high context switch overhead.
Computer Networks

The OSI Model — 7 Layers

The Open Systems Interconnection (OSI) model provides a conceptual framework that standardizes the communication functions of a network into seven layers.

Layer 7 — Application : HTTP, FTP, DNS, SMTP Layer 6 — Presentation : Encryption, Compression, JPEG Layer 5 — Session : Manages sessions (NetBIOS, RPC) Layer 4 — Transport : TCP (reliable), UDP (fast) Layer 3 — Network : IP addressing, Routing (IP, ICMP) Layer 2 — Data Link : MAC addresses, Framing (Ethernet) Layer 1 — Physical : Bits on wire (cables, signals) Mnemonic: "All People Seem To Need Data Processing"

The TCP/IP model collapses OSI's 7 layers into 4: Application, Transport, Internet, and Network Access.

Theory of Computation

Deterministic Finite Automaton (DFA)

A DFA is a 5-tuple M = (Q, Σ, δ, q₀, F) where Q is a finite set of states, Σ is the input alphabet, δ is the transition function, q₀ is the start state, and F is the set of accept states.

DFA that accepts strings ending with '01': States: q0 (start), q1, q2 (accept) Alphabet: {0, 1} Transition Table: State | 0 | 1 ------+----+--- q0 | q1 | q0 q1 | q1 | q2 q2 | q1 | q0 String "001" → q0 -0→ q1 -0→ q1 -1→ q2 ✓ ACCEPTED String "010" → q0 -0→ q1 -1→ q2 -0→ q1 ✗ REJECTED

A NFA allows multiple transitions on the same input and ε-transitions. Every NFA can be converted to an equivalent DFA (Subset Construction).

Q1

What is a process? Differentiate between a process and a program.

Easy

Program: A static set of instructions stored in a file on disk. It is a passive entity.

Process: A program in execution. It is an active entity that includes the program code, current activity (program counter), stack, heap, and data section. Multiple instances of the same program can run as separate processes, each with its own PCB.

🎯 Tip: A common exam question — always mention PCB, states, and resources.

Q2

Consider 3 processes with burst times 24, 3, 3. Calculate average waiting time for FCFS and SJF.

Medium

FCFS (arrival order: P1=24, P2=3, P3=3):

Waiting times: P1=0, P2=24, P3=27 → Avg = (0+24+27)/3 = 17 ms

SJF (sorted: P2=3, P3=3, P1=24):

Waiting times: P2=0, P3=3, P1=6 → Avg = (0+3+6)/3 = 3 ms

🎯 SJF gives optimal average waiting time among non-preemptive algorithms.

Q3

Explain the OSI model with the functions of each layer.

Medium

The OSI (Open Systems Interconnection) model has 7 layers:

1. Physical: Transmits raw bits (cables, hubs).

2. Data Link: MAC addressing, error detection, framing (Ethernet, switches).

3. Network: Logical addressing and routing (IP, routers).

4. Transport: End-to-end delivery, flow/error control (TCP, UDP).

5. Session: Session management, synchronization (NetBIOS).

6. Presentation: Data translation, encryption, compression (SSL/TLS).

7. Application: User interface to network services (HTTP, FTP, DNS).

🎯 Mnemonic: "All People Seem To Need Data Processing" (top to bottom).

Q4

Construct a DFA for the language L = {w ∈ {0,1}* | w ends with '00'}.

Hard

DFA M = (Q, Σ, δ, q0, F) where Q = {q0, q1, q2}, Σ = {0,1}, F = {q2}

Transition Table:

State | 0 | 1
------+----+-----
→q0 | q1 | q0
 q1 | q2 | q0
*q2 | q2 | q0

Explanation: q0 = "haven't seen 0 yet", q1 = "seen exactly one trailing 0", q2 = "seen two or more trailing 0s" (accept state). Any 1 resets to q0.

🎯 Always verify with a few test strings. "100" → q0→q1→q2→q2 ✓

Q5

What is the difference between TCP and UDP? When would you use each?

Easy

TCP (Transmission Control Protocol): Connection-oriented, reliable, ordered delivery, flow control, error correction. Higher overhead. Used for: web (HTTP), email (SMTP), file transfer (FTP).

UDP (User Datagram Protocol): Connectionless, unreliable, no ordering, low overhead. Fast. Used for: live streaming, DNS, VoIP, gaming, video calls.

🎯 Rule: Use TCP when accuracy matters. Use UDP when speed matters more than perfection.

Q6

Explain Banker's Algorithm for deadlock avoidance with a suitable example.

Hard

Banker's Algorithm (Dijkstra) avoids deadlock by simulating resource allocation and checking for a "safe state" before granting requests.

Key Data Structures:

• Available[]: Resources currently available

• Max[][]: Max demand of each process

• Allocation[][]: Currently allocated resources

• Need[][] = Max - Allocation

Safety Algorithm: Find a process whose Need ≤ Available. Simulate it finishing — add its Allocation back to Available. Repeat. If all processes finish, the state is safe.

🎯 This is a 10-mark question favorite. Practice with 5 processes × 3 resource types.
